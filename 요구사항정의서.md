# 시스템 트레이딩 시스템 요구사항 정의서

**프로젝트명:** Exodus Trading System
**작성일:** 2025-10-28
**버전:** 1.2

---

## 1. 개요

### 1.1 문서 목적
본 문서는 시스템 트레이딩 시스템의 기능적/비기능적 요구사항을 정의하고, 시스템 개발을 위한 명확한 가이드라인을 제시한다.

### 1.2 시스템 개요
데이터와 통계에 기반한 퀀트 투자 기법을 활용하여 Buy/Sell/Hold 신호를 생성하고, 한국투자증권 OpenAPI를 통해 주식을 자동으로 매매하는 시스템이다.

### 1.3 범위
- 한국 주식시장 대상
- 한국투자증권 계좌 연동
- 퀀트 투자 전략 기반 자동매매

---

## 2. 시스템 목적 및 범위

### 2.1 시스템 목적
- 체계적이고 일관된 투자 전략 실행
- 감정 배제한 자동화된 매매 의사결정
- 백테스트를 통한 전략 검증 및 최적화
- 실시간 시장 데이터 기반 신속한 매매 실행

### 2.2 목표 사용자
- 퀀트 투자에 관심있는 개인 투자자
- 시스템 트레이딩을 실행하고자 하는 투자자
- 투자 전략을 자동화하고자 하는 사용자

## 3. 기능 요구사항

### 3.1 한국투자증권 OpenAPI 연동 (FR-001)

#### 3.1.1 인증 및 연결 관리
- **FR-001-01:** 한국투자증권 OpenAPI 인증 (APP Key, Secret Key)
- **FR-001-02:** 토큰 발급 및 갱신 자동화
- **FR-001-03:** API 연결 상태 모니터링 및 재연결 처리

#### 3.1.2 시장 데이터 수집
- **FR-001-04:** 실시간 현재가 조회
  - 종목 코드별 현재가, 시가, 고가, 저가, 거래량
  - 전일 대비 등락률

- **FR-001-05:** 시계열 데이터 수집
  - 일봉, 주봉, 월봉 데이터
  - 분봉 데이터 (1분, 5분, 10분, 30분, 60분)
  - OHLCV (시가, 고가, 저가, 종가, 거래량) 데이터

- **FR-001-06:** 기업 기본 정보 조회
  - 시가총액, PER, PBR, ROE 등 재무지표
  - 업종, 섹터 정보

#### 3.1.3 주식 매매 실행
- **FR-001-07:** 시장가 주문 (매수/매도)
- **FR-001-08:** 지정가 주문 (매수/매도)
- **FR-001-09:** 조건부 주문 (손절/익절)
- **FR-001-10:** 주문 체결 확인
- **FR-001-11:** 주문 취소 및 정정
- **FR-001-12:** 미체결 주문 조회

---

### 3.2 백테스트 시스템 (FR-002)

#### 3.2.1 백테스트 실행
- **FR-002-01:** 과거 데이터 기반 전략 시뮬레이션
- **FR-002-02:** 백테스트 기간 설정 (시작일, 종료일)
- **FR-002-03:** 초기 투자금액 설정
- **FR-002-04:** 수수료 및 세금 반영

#### 3.2.2 백테스트 결과 분석
- **FR-002-05:** 수익률 계산 (총 수익률, 연환산 수익률)
- **FR-002-06:** 최대 낙폭 (MDD, Maximum Drawdown) 계산
- **FR-002-07:** 승률 및 손익비 계산
- **FR-002-08:** 샤프 지수 (Sharpe Ratio) 계산
- **FR-002-09:** 거래 횟수 및 평균 보유기간
- **FR-002-10:** 벤치마크 대비 성과 비교 (KOSPI, KOSDAQ)

#### 3.2.3 백테스트 결과 시각화
- **FR-002-11:** 누적 수익률 그래프
- **FR-002-12:** 일별 수익률 분포
- **FR-002-13:** 낙폭 구간 시각화
- **FR-002-14:** 매매 시점 표시 (차트 위 Buy/Sell 마커)

### 3.3 계좌 관리 시스템 (FR-003)

#### 3.3.1 보유 종목 관리
- **FR-003-01:** 현재 보유 종목 조회
  - 종목명, 종목코드, 보유수량
  - 평균 매입가, 현재가
  - 평가손익, 수익률

- **FR-003-02:** 보유 종목 비중 계산
- **FR-003-03:** 종목별 실시간 평가금액 업데이트

#### 3.3.2 거래 이력 관리
- **FR-003-04:** 일별 거래 내역 조회
  - 매수/매도 구분, 종목, 수량, 가격, 시간

- **FR-003-05:** 기간별 거래 이력 조회
- **FR-003-06:** 거래 이력 CSV/Excel 내보내기
- **FR-003-07:** 거래 이력 데이터베이스 저장

#### 3.3.3 손익 관리
- **FR-003-08:** 실현 손익 계산
  - 매도 체결 시 손익 기록

- **FR-003-09:** 미실현 손익 계산
  - 보유 종목의 평가손익

- **FR-003-10:** 일별/월별/연별 손익 집계
- **FR-003-11:** 손익 추이 그래프

#### 3.3.4 계좌 잔액 관리
- **FR-003-12:** 예수금 조회
- **FR-003-13:** 총 평가금액 계산 (예수금 + 보유주식 평가금액)
- **FR-003-14:** 가용금액 계산 (매수 가능 금액)
- **FR-003-15:** 계좌 자산 변동 추이 기록

### 3.4 투자 전략 관리 (FR-004)

#### 3.4.1 전략 유형
- **FR-004-01:** 모멘텀 추종 전략
  - 이동평균선 (SMA, EMA)
  - 이동평균 수렴확산 (MACD)
  - 상대강도지수 (RSI)
  - 볼린저 밴드
  - 가격 모멘텀 (N일 수익률)

- **FR-004-02:** 평균 회귀 전략
  - 평균 회귀 지표
  - 과매수/과매도 구간 식별
  - 밸류에이션 기반 매매

- **FR-004-03:** 팩터 투자 전략
  - 가치 팩터 (PER, PBR, PCR)
  - 모멘텀 팩터 (12개월 수익률)
  - 퀄리티 팩터 (ROE, 영업이익률)
  - 저변동성 팩터
  - 멀티팩터 조합

#### 3.4.2 전략 설정 및 관리
- **FR-004-04:** 전략별 파라미터 설정
  - 이동평균선 기간
  - RSI 임계값
  - 팩터 가중치
  - 리밸런싱 주기

- **FR-004-05:** 전략 저장 및 불러오기 (JSON/YAML)
- **FR-004-06:** 전략 버전 관리
- **FR-004-07:** 다중 전략 동시 운용
- **FR-004-08:** 전략별 투자 비중 할당

#### 3.4.3 리스크 관리
- **FR-004-09:** 손절매 설정 (손실률 기준)
- **FR-004-10:** 익절매 설정 (수익률 기준)
- **FR-004-11:** 포지션 크기 조절 (Kelly Criterion, Fixed Fractional)
- **FR-004-12:** 최대 종목 수 제한
- **FR-004-13:** 종목당 최대 투자 비중 제한
- **FR-004-14:** 섹터/업종 분산 설정

### 3.5 매매 신호 생성 (FR-005)

#### 3.5.1 신호 생성
- **FR-005-01:** 전략 기반 Buy 신호 생성
  - 조건 충족 시 매수 신호 발생
  - 신호 강도 (Confidence Score)

- **FR-005-02:** 전략 기반 Sell 신호 생성
  - 조건 충족 시 매도 신호 발생
  - 손절/익절 조건 포함

- **FR-005-03:** Hold 신호 (매매 조건 미충족)

#### 3.5.2 신호 관리
- **FR-005-04:** 신호 발생 이력 기록
  - 종목, 신호 유형, 발생 시간, 가격, 조건

- **FR-005-05:** 신호 알림 (이메일, 메시지, 푸시)
- **FR-005-06:** 신호 우선순위 설정
- **FR-005-07:** 신호 필터링 (최소 거래량, 가격 범위)

#### 3.5.3 신호 검증
- **FR-005-08:** 실시간 신호 정확도 모니터링
- **FR-005-09:** 신호 성과 분석 (신호 발생 후 수익률 추적)

  - 주문 가격 결정 (시장가/지정가)

- **FR-006-11:** 긴급 전량 매도 (패닉 버튼)
- **FR-006-12:** 수동 주문 실행 (자동매매와 별개)

#### 3.6.4 안전장치
- **FR-006-13:** 일일 최대 거래 횟수 제한
- **FR-006-14:** 일일 최대 손실 한도 설정
- **FR-006-15:** 비정상 신호 필터링 (급등락 종목 제외)
- **FR-006-16:** API 오류 시 자동매매 중단

### 3.7 사용자 인증 및 권한 관리 (FR-007)

#### 3.7.1 사용자 인증
- **FR-007-01:** JWT 기반 사용자 인증
  - Access Token 발급 (유효기간: 1시간)
  - Refresh Token 발급 (유효기간: 7일)
  - Token 갱신 메커니즘

- **FR-007-02:** 사용자 회원가입
  - 이메일 인증
  - 비밀번호 암호화 저장 (bcrypt)
  - 사용자 정보 검증

- **FR-007-03:** 사용자 로그인/로그아웃
  - 로그인 시 JWT 토큰 발급
  - 로그아웃 시 토큰 무효화
  - 다중 기기 로그인 지원

#### 3.7.2 권한 관리
- **FR-007-04:** 역할 기반 접근 제어 (RBAC)
  - Admin: 전체 시스템 관리
  - User: 개인 계좌 및 전략 관리
  - Viewer: 읽기 전용 접근

- **FR-007-05:** API 엔드포인트 권한 제어
  - 인증된 사용자만 접근 가능
  - 역할별 접근 권한 검증

#### 3.7.3 보안
- **FR-007-06:** 비밀번호 정책
  - 최소 8자 이상, 영문/숫자/특수문자 조합
  - 비밀번호 재설정 기능

- **FR-007-07:** 세션 관리
  - 자동 로그아웃 (30분 비활성 시)
  - 동시 세션 수 제한

- **FR-007-08:** API Rate Limiting
  - 사용자별 요청 제한 (분당 100회)
  - IP 기반 제한 (DDoS 방지)

## 4. 비기능 요구사항

### 4.1 성능 요구사항 (NFR-001)
- **NFR-001-01:** 실시간 데이터 수집 지연시간 1초 이내
- **NFR-001-02:** 매매 신호 생성 후 주문 실행 3초 이내
- **NFR-001-03:** 동시에 100개 이상 종목 모니터링 가능
- **NFR-001-04:** 백테스트 실행 시간 (10년 데이터 기준) 5분 이내

### 4.2 안정성 요구사항 (NFR-002)
- **NFR-002-01:** 시스템 가용성 99.5% 이상 (장 운영 시간 기준)
- **NFR-002-02:** API 연결 실패 시 자동 재연결
- **NFR-002-03:** 에러 발생 시 자동매매 중단 및 알림
- **NFR-002-04:** 데이터 백업 (일 1회 이상)

### 4.3 보안 요구사항 (NFR-003)
- **NFR-003-01:** JWT 기반 인증 시스템 구현
  - Access/Refresh Token 분리
  - Token 만료 시간 관리
  - HTTPS 통신 강제

- **NFR-003-02:** API 인증 정보 암호화 저장
- **NFR-003-03:** 민감 정보 환경 변수 또는 보안 저장소 사용
- **NFR-003-04:** 거래 이력 데이터 암호화
- **NFR-003-05:** 로그에 민감 정보 기록 금지
- **NFR-003-06:** CORS 정책 설정 (허용된 도메인만 접근)
- **NFR-003-07:** SQL Injection, XSS 등 보안 취약점 방어

### 4.4 확장성 요구사항 (NFR-004)
- **NFR-004-01:** 새로운 투자 전략 추가 용이성
- **NFR-004-02:** 다른 증권사 API 연동 가능 구조
- **NFR-004-03:** 플러그인 방식 전략 모듈 지원

### 4.5 사용성 요구사항 (NFR-005)
- **NFR-005-01:** 웹 기반 사용자 인터페이스 제공
  - 반응형 디자인 (모바일, 태블릿, 데스크톱 지원)
  - 직관적인 UI/UX
  - 실시간 데이터 업데이트

- **NFR-005-02:** 설정 파일 기반 시스템 구성
- **NFR-005-03:** 명확한 로그 메시지 및 에러 메시지
- **NFR-005-04:** 다크 모드 지원
- **NFR-005-05:** 다국어 지원 (한국어, 영어)

### 5.2 기술 스택

- **Framework:** React 18+ with Next.js 14+ (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **UI Components:** shadcn/ui
- **State Management:** Zustand or Redux Toolkit
- **Data Fetching:** TanStack Query (React Query)
- **Charts:** Recharts or Apache ECharts
- **Real-time:** Socket.io Client or WebSocket API

#### Backend
- **Framework:** FastAPI (Python 3.11+)
- **Authentication:** JWT (PyJWT, python-jose)
- **Password Hashing:** bcrypt or passlib
- **API Documentation:** OpenAPI (Swagger) - FastAPI 내장
- **Data Processing:** pandas, numpy
- **Technical Analysis:** TA-Lib
- **Task Scheduling:** APScheduler or Celery + Redis
- **WebSocket:** FastAPI WebSocket or Socket.io
- **Testing:** pytest, pytest-asyncio
- **Logging:** Loguru or structlog

#### Database
- **Primary DB:** PostgreSQL 15+
  - User data, trading history, strategies
  - Time-series data with TimescaleDB extension (Optional)
- **Cache:** Redis 7+ (Session, Rate Limiting, Task Queue)
- **ORM:** SQLAlchemy 2.0+ with Alembic (Migration)

#### Infrastructure
- **Containerization:** Docker
- **Orchestration:** Docker Compose
- **Web Server:** Nginx (Reverse Proxy, Static Files)
- **Process Manager:** Gunicorn + Uvicorn Workers (FastAPI)

#### DevOps & Tools
- **Version Control:** Git
- **CI/CD:** GitHub Actions or GitLab CI (선택적)
- **Environment Management:** python-dotenv
- **Code Quality:**
  - Backend: Black, Flake8, mypy
  - Frontend: ESLint, Prettier
- **Monitoring:** Prometheus + Grafana (선택적)

## 10. 리스크 및 대응 방안

### 10.1 기술적 리스크
- **API 장애:** 재연결 로직, 대체 데이터 소스
- **데이터 손실:** 정기 백업, 데이터 검증, Docker Volume 영구 저장
- **버그로 인한 오주문:** 철저한 테스트, Dry-run 모드
- **컨테이너 장애:** Health Check, 자동 재시작 정책
- **성능 저하:** Redis 캐싱, 데이터베이스 인덱싱, 쿼리 최적화

### 10.2 보안 리스크
- **인증 취약점:** JWT 토큰 탈취 방어, HTTPS 강제, Refresh Token Rotation
- **SQL Injection:** ORM 사용, Prepared Statement
- **XSS 공격:** Input Validation, Output Encoding
- **DDoS 공격:** Rate Limiting, IP 기반 차단

### 10.3 운영 리스크
- **전략 실패:** 백테스트 검증, 점진적 자금 투입
- **시장 급변:** 손절매 설정, 일일 손실 한도
- **예상치 못한 비용:** 수수료 계산 정확성, 슬리피지 고려
- **시스템 다운타임:** Docker Compose로 빠른 복구, 자동 재시작
